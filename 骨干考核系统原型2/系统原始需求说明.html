骨干考核系统

参赛题目：

骨干考核系统


基本功能：

1.     基本信息管理：学员基本信息，学员学分情况等基本信息管理

2.     培养计划管理：实现培养计划，分值等所有加分减分项目管理

3.     打分管理：实现学员每项考核具体打分，分值作为是否骨干主要依据

4.     评价管理：实现导师、HEAD、客户等关键角色对学员评价

5.     出入骨干池管理：更新最终学员是否成为骨干

（可考虑手动或自动方式实现）

6.     批量信息录入：实现骨干相关信息的批量录入，提高使用者工作效率

7.     骨干明细查看：可按照不同维度查看骨干数量、分布及详情

8.     权限管理：系统至少区分管理员，信息录入员和普通用户三种权限


参赛要求：

1.     每个团队3—5人

2.     参赛周期约一个月左右

3.     所有参赛团队绿区开发（方便选手利用工作之余完善作品）

4.     语言不做限制，数据库可使用mysql、mongodb


验收条件：

1.     提供完整代码包

2.     提供基本操作说明

3.     提供系统设计文档

（包含设计思路、架构选型，性能、稳定性评估，开发和部署环境要求）


验收方式：

1.     按照阶段输出相应成果参加评比：

初期（30分） + 中期（70分） = 中期结果

终期结果不参考前期分值

2.     技术评委分组按照侧重点验收：

架构及代码质量：吕兴，张强，张海强

数据库及扩展性：马涛，代林青

功能验收全体参与

3.     技术评委和业务功能评委具体评分比例：

各自占比50%

4.     比赛阶段无淘汰制，所有参赛团队不允许中途弃赛


评分标准：

初期验收（总分值30分）

1.     技术选型及架构文档（10分）

2.     基础框架搭建（15分）

3.     团队分工计划（5分）

中期验收（总分值70分）

1.     页面展示（10分）

2.     功能点（共30分）：

1（3分），2（4分），3（4分），4（4分）

5（4分），6（4分），7（3分），8（4分）

3.     代码质量（10分）

4.     数据库设计（10分）

5.     基础文档（共10分）：

架构设计文档（7分），操作说明（3分）

终期验收（总分值100分）

1.     整体页面展示（10分）

2.     功能点（共30分）：

1（3分），2（4分），3（4分），4（4分）

5（4分），6（4分），7（3分），8（4分）

3.     加分项（20分）

4.     代码架构及质量（15分）

5.     数据库设计（15分）

6.     基础文档（共10分）：

架构设计文档（7分），操作说明（3分）


数据库得分标准

Mysql

1. 数据结构5分，表，字段，主外键合理性

2. 表结构可扩展性3分

3. SQL语句5分，通用性，结构合理性

4. SQL性能，安全2分

Mongodb

1. 数据结构 collection，属性，索引合理性 5分

2. collection可扩展性 3分

3. 语句通用性，合理性5分

4. 并发和安全 2分


其他说明：

1.     比赛时间分布：初期25%，中期60%，终期15%

（具体时间点按照比赛天数换算最终值）

2.     加分项功能不作具体要求，比如增加日志管理、各类报表展示等非必须功能，

      也可在当前基础功能做优化展示所开发系统亮点

3.     每个部门在绿区申请笔记本单独部署演示（以免部署演示互相影响耗时）

4.     大赛除基本系统要求外，另附一份基础数据样例，比赛过程中业务相关问题找团队

        对应评委沟通解答（对应评委分布依照参赛团队划分）

5.     后续选择最优系统继续完善以供正式使用
